Another situation ive ran into is the two ways of using props (props) & ({name})

one is a specifier, the other a whole package

Specifier way of using props:
const Button = ({ children, onClick }) => {
  return (
    <button onClick={onClick}>
      {children}
    </button>
  );
};

export default Button;



whole package of using props:
export default function Button(props) {
    return (
        <button onClick={props.onClick}>{props.children}</button>
    )
}

Why do you need children?

Because otherwise the content inside <Button>...</Button> would never appear.
Without children your output would be Empty button. <button></button>
The icon and text are lost because you never rendered children.
