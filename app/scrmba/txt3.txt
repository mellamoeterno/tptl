Answer:




The point is actually not:




"Am I using find()?"




The point is:


"Do I want my function to return the result of find()?"






Case 1: Yes, use return


function getPizzaDetail(id: number) {
    return menu.find(pizza => pizza.id === id)
}

const pizza = getPizzaDetail(2)



getPizzaDetail() is supposed to give the caller a pizza, 
so you return the result of find().






Case 2: No, don't use return

Suppose you only want to log the pizza:

function printPizza(id: number) {
    const pizza = menu.find(pizza => pizza.id === id)
    console.log(pizza)
}

printPizza(2)



No return is needed because the function's job is to print 
something, not hand a value back.






SO basically i have to watchout if the code inside the function or const is 
doing something, or producing a value.




BUT ALSO .find DOESNT AUTOMATICALLY RETURN THE VALUE



So instead of memorizing every method, ask(Ai probably most times):

Can I assign its result to a variable?

const filtered = array.filter(...)
const exists = array.includes(...)
const text = name.toUpperCase()

If that makes sense, the method is producing a useful return value.
