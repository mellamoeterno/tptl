In here theres a usage of .find 


.find() calls your callback on each item, but it inspects the return value. 

If your callback returns true, .find() immediately stops searching and hands you 
that item. 



**                                  {
    const order = orderQueue.find(
        orderObj => orderObj.id === orderId
    )

    order.status = "completed"

    return order
}**


in that part is .find triggering order.status and return after getting the value true? sorta like a if statement?

yes:

.find() is literally a loop with an if statement inside it. 
The callback orderObj => orderObj.id === orderId is just the condition for that if statement. 


When that condition becomes true, .find() "triggers" and returns that item.

so its sort of working like a if statement without {}
