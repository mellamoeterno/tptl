Working with images asynchronously



const image = new Image()
image.src = "http://..."



This image constructor above seems to be a good way for the case of
pre loading images, why is that?


What happens?


new Image() creates an HTMLImageElement in memory.

Setting image.src tells the browser:

"Start fetching this image from the server."

The browser downloads it and stores it in its image cache.
The image does not have to be visible on the page for the download to happen.
Why is this useful?


Imagine you have a slideshow.

Without preloading:

nextImage.src = "slide2.jpg"

When the user clicks "Next":

User clicks
      ↓
Browser starts downloading
      ↓
User waits...
      ↓
Image finally appears

The user sees a delay or a blank area.



With preloading:

const preload = new Image()
preload.src = "slide2.jpg"

This happens as soon as the page loads:
