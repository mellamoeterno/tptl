Promise


a promise seems to be a state of data in which it hasnt been 'delivered' yet.




--

.catch()

handle promise rejection so we have the capability to do some code 
(like showing a message to the user) 
(or to try a different api)

work similarly to .then()


--


.finally()

also work similarly to .then()

And its parameter will run at the end of the process wether the promise 
rejected or resolved. (nothing to do with error handling, this is just to know
.finally)

finally method is useful in some situtions where you need some code to execute at
the end of the Asynchronous operation.


---

We also want to handle a promise rejection with async await syntax,
and for this we need try, catch blocks.

try {
    // the code we want to try to execute
} catch(err) {
    // the code to execute on an error
} finally {
    // code to execute at the end of the operation
}
